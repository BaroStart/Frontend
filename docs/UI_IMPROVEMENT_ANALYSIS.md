# UI 개선 검토 및 제안

> 과제 페이지 아이콘, 마이페이지 달성률 색상, 뱃지 디자인 검토

---

## 1. 과제 페이지 — 아이콘 제거 및 레이아웃 제안

### 1.1 현재 문제점

- **정보 전달 대비 비중 과다**: 과목명 텍스트가 이미 있는데, 아이콘은 같은 정보를 재표현하며 시각적 노이즈만 증가
- **시선 분산**: 브랜드 컬러 아이콘이 과제 제목보다 먼저 눈에 들어와 핵심 정보(제목·상태) 인지가 느려짐
- **여백 부족**: 48×48px 아이콘 영역으로 인해 텍스트 영역이 상대적으로 좁아짐

### 1.2 아이콘 제거 시 장점

| 항목 | 설명 |
|------|------|
| **정보 위계 명확화** | 과목·제목·설명·날짜가 텍스트 위계만으로 정리되어 읽기 쉬움 |
| **미니멀 톤** | Notion·인스타 스타일의 깔끔한 리스트 향상 |
| **레이아웃 일관성** | 다양한 과목 아이콘 스타일 차이 제거로 전체가 더 정돈됨 |
| **유지보수** | 과목 추가/변경 시 아이콘 매핑 없이 텍스트만 관리 가능 |

### 1.3 아이콘 없는 레이아웃 제안

#### A안: 라벨 + 제목 중심 구조 (추천)

```
┌─────────────────────────────────────────────────────────┐
│  국어          [완료] 또는 [미완료]                      │
│  영어 독해 - Chapter 5 문제풀이                          │
│  교재: 수능특강 영어독해 / 페이지: 45-52                  │
│  2026.02.02 오전 10:45 제출 완료                        │
└─────────────────────────────────────────────────────────┘
```

- 과목: `text-[11px] uppercase tracking-wider text-slate-400` (작고 보조)
- 상태 태그: 우측 상단, 브랜드(완료) / 회색(미완료)
- 제목: `text-base font-bold text-slate-900` (가장 강조)
- 설명: `text-sm text-slate-600`
- 날짜: `text-[11px] text-slate-400`

#### B안: 과목 컬러 라인 (선택)

```
┌─────────────────────────────────────────────────────────┐
│ ┃ 국어 · 영어 독해 - Chapter 5 문제풀이      [완료]     │
│ ┃ 교재: 수능특강 영어독해...                             │
│ ┃ 2026.02.02 제출 완료                                  │
└─────────────────────────────────────────────────────────┘
```

- 좌측 3–4px 세로 라인만 과목별 색상으로 구분
- 아이콘 대신 얇은 라인으로 과목 식별 가능

#### C안: 제목 + 상태 원형 인디케이터

```
┌─────────────────────────────────────────────────────────┐
│  영어 독해 - Chapter 5 문제풀이              ● 완료      │
│  국어 · 교재: 수능특강...                               │
│  2026.02.02 제출 완료                                   │
└─────────────────────────────────────────────────────────┘
```

- 상태만 작은 원형(●) + 텍스트로 표현
- 아이콘 없이도 진행 상태를 직관적으로 전달

### 1.4 결론

- **아이콘 제거 권장**: 과목·제목·상태·날짜만으로도 정보 전달이 충분함
- **우선 적용**: A안, 이후 필요 시 B안(과목 컬러 라인) 추가 검토

## 2. 마이페이지 — 과목별 달성률 색상 지정 UX

### 2.1 현재 한계

- 링 차트·과목명·퍼센트가 모두 `hsl(var(--brand))`로 고정됨
- 사용자 취향·서비스 톤 변경 시 과목별 색상을 바꾸기 어려움

### 2.2 과목별 색상 선택 UX 검토

#### 가능 여부: **가능**

| 요소 | 구현 난이도 | 비고 |
|------|-------------|------|
| 설정에서 과목별 색 선택 | 중 | localStorage + 컬러 피커 |
| 서비스 톤(테마) 변경 | 중 | CSS 변수 기반 테마 |
| 프리셋 팔레트 제공 | 하 | 4–6개 톤 제공 |

#### 구현 방향

1. **과목별 색상 설정**
   - `SubjectAchievementItem`에 `color?: string` 추가
   - 기본값: `--brand` 또는 과목별 프리셋(국어: 청록, 영어: 파랑, 수학: 보라 등)
   - 설정 화면에서 컬러 피커 또는 팔레트로 변경

2. **저장**
   - `localStorage` 키: `subjectColors` 또는 `userPreferences`
   - `{ '국어': '193 87% 40%', '영어': '220 70% 50%', ... }` 형태

3. **UI**
   - 마이페이지 섹션 헤더에 "색상 편집" 버튼
   - 클릭 시 모달/드로어에서 과목별 색상 선택

#### 리스크 및 주의사항

| 항목 | 내용 |
|------|------|
| 접근성 | 색상 대비·색맹 고려, 텍스트/아이콘으로도 구분 가능하도록 |
| 일관성 | 달력·과제 등 다른 화면과 색상 매핑 유지 여부 결정 필요 |
| 단순화 | 처음에는 3가지 프리셋(클래식/파스텔/비비드)만 제공하는 것도 방법 |

### 2.3 우선순위 제안

- **Phase 1**: 과목별 고정 프리셋(국어·영어·수학 각각 다른 색) 적용
- **Phase 2**: 설정에서 과목별 색상 변경 기능 추가

## 3. 마이페이지 — 뱃지 디자인 개선 방향

### 3.1 현재 한계

- 원형·둥근 사각형 혼용으로 인한 일관성 부족
- 잠김/획득 구분은 있으나, "성취 보상" 느낌이 약함
- 획득 시에도 단순한 솔리드 색상으로 다소 평면적

### 3.2 디자인 개선 방향

#### A. 상태 표현 강화

| 상태 | 현재 | 개선 방향 |
|------|------|-----------|
| 획득 | 브랜드 배경 + 흰색 아이콘 | 그라데이션, 은은한 glow, 작은 장식 요소 |
| 잠김 | 회색 점선 테두리 | 반투명 + 약한 텍스처, 아이콘 실루엣만 표현 |

#### B. 형태 차별화 (의미 기반)

| 뱃지 유형 | 예시 | 형태 | 컨셉 |
|-----------|------|------|------|
| 마스터 | 국어/영어/수학 마스터 | 육각형, 방패형 | "등급" 느낌 |
| 출석/연속 | 7일·30일 출석 | 원형 | "연속성" |
| 첫 달성 | 첫 과제, 100시간 | 별/다이아몬드형 | "마일스톤" |
| 루틴 | 아침 루틴, 포모도로 | 둥근 사각형 | "습관" |

#### C. 획득 시 시각적 강조

1. **그라데이션**: `linear-gradient(135deg, brand 0%, brand-light 100%)`
2. **은은한 glow**: `box-shadow: 0 0 12px hsl(var(--brand)/30)`
3. **미세한 립(테두리)**: inner border로 금속감 표현
4. **애니메이션**: 획득 시 짧은 스케일/펄스 애니메이션

#### D. 잠김 상태 개선

1. **아이콘 실루엣**: 자물쇠 대신 해당 뱃지 아이콘을 회색 실루엣으로
2. **그레이스케일 + 흐림**: `grayscale(1) opacity-60`
3. **점선 테두리 유지**: 잠김임을 명확히 전달

### 3.3 뱃지 유형별 형태 매핑 제안

```
마스터(국어/영어/수학 등)     → rounded-2xl (카드형)
출석/연속(7일, 30일 등)      → rounded-full (원형)
마일스톤(첫 과제, 100시간)   → 별/다이아몬드 (clip-path)
루틴(아침, 포모도로 등)      → rounded-xl (중간)
```

### 3.4 우선 적용 제안

1. **획득 강조**: 그라데이션 + 은은한 glow
2. **잠김 구분**: 아이콘 실루엣 + `grayscale` + `opacity`
3. **형태**: 현재 `shapeFor` 로직 유지·보완으로 단계적 적용
