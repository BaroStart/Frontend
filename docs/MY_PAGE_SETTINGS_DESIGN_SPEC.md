# 마이페이지 설정 및 개인화 디자인 명세

> 컬러 팔레트 / 설정 UI / 프로필 사진 / 뱃지 보상 화면 — 디자인 관점 검토

---

## 1. 목표

홈페이지의 타임테이블 색상 설정 UX와 동일한 흐름을 유지하면서,
사용자가 전체 서비스 컬러 테마와 뱃지를 직접 커스터마이징할 수 있는 설정 화면을 설계한다.

- 사용자는 **"내가 바꿨다"**는 감각을 명확히 인지해야 한다.
- 백엔드 없이도 설정이 유지되는 구조를 전제로 한다.

---

## 2. 컬러 팔레트 설정 UI

### 2.1 컬러 시스템 방향

- 차분하고 신뢰감 있는 인상을 유지하면서 개인화 가능하도록 설계
- 프리셋 팔레트 + (선택) 커스텀 컬러 피커

### 2.2 프리셋 팔레트 구성

| 프리셋 | 설명 |
|--------|------|
| 기본 | 에메랄드 그린 계열 (메인) |
| 파스텔 | 부드러운 파스텔 톤 |
| 차분 | 어두운 그레이·남색 |
| 비비드 | 선명한 포인트 컬러 |
| 소프트 민트 | 민트 그린 계열 |

### 2.3 각 팔레트 구성

- **대표 컬러 1개** + **서브 컬러 2~3개**
- 작은 컬러 칩 (circle 또는 rounded square) 형태로 미리보기 제공

### 2.4 선택 상태 표현

현재 선택된 팔레트는:

- **테두리 강조** (`border-2 border-slate-800`)
- **체크 아이콘** 표시
- 시각적으로 즉시 구분 가능해야 함

### 2.5 기본 컬러 인식 문제 보완

**문제점**: 앱 진입 시 기본 컬러가 이미 선택된 상태로 보이며,
사용자가 기본값인지, 직접 선택한 값인지 혼동할 수 있음

**해결 방식**:

- "기본 (Default)" 라벨 명시
- 또는 "현재 적용 중" 텍스트 노출
- 사용자의 직접 선택 이후에는 해당 표시 제거 또는 변경

## 3. 커스텀 컬러 선택 기능 (선택사항)

### 3.1 기능 개요

- 프리셋 외에 사용자 지정 컬러 선택 가능
- 기본 UX 흐름을 방해하지 않음

### 3.2 컬러 피커 UI

- 심플한 팔레트 아이콘 사용
- 설정 페이지 내부에서만 노출
- 서브 옵션으로 제공

### 3.3 적용 범위

- 타임테이블
- 버튼 포인트 컬러
- 강조 UI, 선택 상태, 활성 상태
- 뱃지 테두리 등

---

## 4. 데이터 저장 방식 (백엔드 없음)

- 컬러 및 설정 정보는 클라이언트에서만 관리
- **저장**: localStorage 또는 IndexedDB
- **동작**: 새로고침 / 재접속 시 유지, 로그아웃 시 초기화 가능

---

## 5. 마이페이지 설정 구조

### 5.1 설정 진입 구조

```
마이페이지 > 설정(Settings)
```

### 5.2 설정 항목 구성

| 섹션 | 항목 |
|------|------|
| 테마 & 컬러 | 컬러 테마 (프리셋), 커스텀 컬러 (선택) |
| 프로필 | 프로필 사진 변경 (로컬) |
| 뱃지 커스터마이징 | 색상, 그라데이션, 테두리 스타일 |
| 기타 | 로그아웃, 상담 버튼 등 |

### 5.3 설정 아이콘

- 톱니바퀴 (`Settings`) 아이콘
- 마이페이지 우측 상단에 고정 배치

## 6. 뱃지 커스터마이징 & 획득 화면 UX

### 6.1 뱃지 커스터마이징

- 뱃지는 단순한 "획득 표시"가 아닌 **개인 아이덴티티 요소**
- 커스터마이징 가능 요소:
  - 색상 변경
  - 배경 그라데이션 선택
  - 테두리 스타일 변경
- 뱃지 컬러는 현재 선택된 전체 테마 팔레트와 자연스럽게 연동

### 6.2 뱃지 획득 화면 개선

| 구분 | 권장 |
|------|------|
| 단순 리스트 | ❌ |
| 카드형 / 모달 / 애니메이션 기반 UX | ✅ |

### 6.3 획득 시 연출

- 축하 애니메이션 제공
- 현재 테마 컬러를 활용한 빛나는 효과
- "대표 뱃지로 설정하기" CTA 버튼 제공

---

## 7. 전체 디자인 톤 & 스타일

- 과하지 않은 **미니멀 + 소프트 UI**
- iOS / Notion 계열의 감성
- 둥근 모서리와 넉넉한 여백 사용

### 7.1 컬러 사용 원칙

- 한 화면에 **1~2개의 포인트 컬러**만 강조
- 나머지는 중립 톤 유지

---

## 8. 기존 기술 명세 (참고)

### 8.1 전역 CSS 변수

| 변수 | 용도 |
|------|------|
| `--brand` | 메인 포인트 컬러 |
| `--brand-light` | 연한 배경 |
| `--brand-medium` | 중간 톤 |

### 8.2 적용 범위

- 달성률 바, 캘린더, 뱃지, 버튼 포인트, 강조 UI

### 8.3 UI 권장안

- 팔레트 카드: 2열 그리드, 각 카드에 3개 스와치 + 이름
- 스와치 형태: `h-4 w-4 rounded-sm` 또는 `rounded-full`
- 적용 피드백: 200ms transition
