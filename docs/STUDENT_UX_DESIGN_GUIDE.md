# 학생용 학습 웹앱 UX/UI 디자인 가이드

> "관리 툴"이 아닌 "공부 메이트 앱"으로

---

## 1. 현재 디자인이 'AI 같아 보이는 이유' (UX 관점 분석)

### 1.1 시각적 패턴

| 현상 | AI 스타일의 특징 | 현재 앱에서의 징후 |
|------|------------------|-------------------|
| **그라데이션 일관성** | 135deg, hsl 기반 그라데이션이 여러 곳에서 반복 | 타임테이블 블록, brand 색상, 카드 등 전역적으로 유사한 패턴 |
| **둥근 모서리 과다** | rounded-2xl, rounded-3xl 남용 | 버튼·카드·배지·모달이 모두 강한 radius |
| **부드러운 그림자** | soft, soft-lg 등 미세한 그림자 | 모든 요소에 shadow-soft 적용 |
| **색상 팔레트** | 파스텔 + brand 청록색 조합 | hsl(193...) 계열 위주, 차별화 부족 |
| **레이아웃** | 카드/리스트 중심의 단순 구조 | 기능 위주 배치, 여백·정렬 불일치 |

### 1.2 감정적 결핍

- **피드백 부족**: 완료, 체크, 기록 시 시각적/촉각적 반응이 약함
- **성취 표현 단순**: 배지·진행률이 정적으로만 보임
- **개인화 부재**: 사용자가 선택할 수 있는 요소가 거의 없음
- **일관된 톤**: 딱딱한 "관리 화면" 느낌, 학생 친화적 어조 부족

### 1.3 구조적 문제

- 컴포넌트별 `px-4`, `p-4`, `gap-2` 등 값이 제각각
- 폰트 크기·굵기 hierarchy가 불명확
- 탭바·헤더·카드 간 시각적 위계가 약함

## 2. 학생 대상 서비스 디자인 컨셉 2~3가지

### 컨셉 A: "스터디 데스크" (Study Desk)

- **감성**: 내 책상 위에 올려둔 플래너, 스티커 메모
- **색상**: 크림/베이지 베이스 + 포인트 컬러 1~2개
- **형태**: 직각·미세 radius (4~8px), 얇은 테두리, 노트 느낌
- **적용**: 메인 배경을 연한 크림색, 카드를 흰색/아이보리로, 강조는 단색 블록

### 컨셉 B: "미니멀 스탠드" (Minimal Stand)

- **감성**: Notion/Linear 스타일의 깔끔한 생산성 앱
- **색상**: 단색(grayscale) + 강조색 1개만
- **형태**: 직선, 얇은 구분선, 여백 위주
- **적용**: 회색 톤 베이스, 완료/액션에만 브랜드 컬러, 타임테이블은 단색 블록

### 컨셉 C: "게이미피케이션 라이트" (Gamification Light)

- **감성**: 플레잉 카드, 캐릭터, 배지 중심
- **색상**: 다채로우되 채도 낮게 (muted)
- **형태**: 아이콘·배지 비중 확대, 마이크로 인터랙션 강화
- **적용**: 완료 시 짧은 애니메이션, 배지 획득 시 전체 화면 축하, 타임테이블 색상 선택

## 3. 컴포넌트 정렬/간격/크기 기준 (디자인 시스템)

### 3.1 간격 스케일 (8px 베이스)

```
space-0: 0
space-1: 4px
space-2: 8px
space-3: 12px
space-4: 16px
space-5: 20px
space-6: 24px
space-8: 32px
space-10: 40px
```

- **영역 내부**: `space-4` (16px)
- **섹션 간**: `space-6` (24px)
- **페이지 horizontal padding**: `16px` (모바일) / `24px` (태블릿)

### 3.2 폰트 스케일

| 용도 | 크기 | 굵기 |
|------|------|------|
| 페이지 타이틀 | 18px | 700 |
| 섹션 타이틀 | 14px | 600 |
| 카드 제목 | 15px | 700 |
| 본문 | 14px | 400 |
| 보조 텍스트 | 12px | 500 |
| 캡션 | 11px | 400 |

### 3.3 컴포넌트 크기

| 요소 | 높이 | radius |
|------|------|--------|
| 버튼 (primary) | 44px | 10px |
| 버튼 (icon) | 40px | 9999px |
| 카드 | auto | 12px |
| 입력 필드 | 48px | 10px |
| 배지/아이콘 | 56px | 50% |

### 3.4 정렬 규칙

- **좌측 정렬**: 제목, 리스트, 본문
- **가운데**: 배지 그리드, 빈 상태
- **우측 정렬**: 액션 버튼, 숫자(시간 등)

## 4. 체류 시간을 늘릴 수 있는 화면 구성 아이디어

1. **홈 첫 화면 훅**: "오늘의 한 마디" + 오늘 완료한 첫 항목 시각적 강조
2. **타임테이블 색상 선택**: 과목/할 일별 색상을 사용자가 선택 → 타임테이블이 "내 것"처럼 느껴짐
3. **배지 획득 모달**: 배지 오픈 시 전체 화면 축하 + 배지가 날아다니는 애니메이션
4. **스트릭 시각화**: 연속 학습 일수火焰 아이콘 + "N일 연속!" 강조
5. **완료 시 마이크로 피드백**: 체크 시 짧은 스케일+페이드, 타임테이블 블록 생성 시 슬라이드 인
6. **주간 요약 카드**: "이번 주 12시간 공부했어요" 등 한 줄 요약 + 그래프

## 5. 실제 적용 가능한 UI 개선 포인트 예시

### 5.1 타임테이블 색상 선택 기능

**현재**: `title` + `id` 해시로 6가지 고정 그라데이션 부여

**개선**:
- 설정 또는 타임테이블 상단에 "색상 선택" 버튼
- 팔레트(8~12색)에서 사용자가 과목/할 일 타입별 색상 지정
- localStorage에 저장 후 타임테이블 블록에 적용

```
UI: [타임테이블] 우측 상단 "색상" 아이콘 클릭 → 팔레트 모달
저장: { "task": "#4A90D9", "assignment": "#E85D75", ... }
```

### 5.2 배지 획득 시 전체 화면 축하 + 날아다니는 애니메이션

**현재**: BadgeSection에서 획득 여부만 표시

**개선**:
- 새 배지 획득 시 전역 이벤트 발행
- 전체 화면 오버레이 + 획득한 배지 아이콘 3~5개가 화면 위를 떠다니는 애니메이션
- 2~3초 후 페이드 아웃

```
애니메이션: 
- 배지 아이콘이 random 위치에서 등장
- float + 약간의 rotate, 각기 다른 딜레이
- keyframes: badge-fly-in, badge-float
```

### 5.3 체크/완료 시 마이크로 인터랙션

- TodoCard 체크: `scale(0.95) → scale(1)` + 체크 아이콘 scale-in
- 타임테이블 블록 추가: `opacity 0 → 1` + `translateY(8px) → 0`

### 5.4 그라데이션 축소

- 타임테이블 블록: 그라데이션 → 단색 또는 매우 연한 2톤
- 버튼: 단색 배경 + 호버 시 `brightness`만 조절

### 5.5 일관된 간격

- MenteeMainPage: `px-4` → `px-4` 유지, 내부 섹션 `gap-6`
- 카드 그룹: `space-y-3` → `space-y-4`로 통일

## 6. 구현 우선순위 제안

| 순위 | 항목 | 효과 | 난이도 |
|------|------|------|--------|
| 1 | 타임테이블 색상 선택 | 개인화·체류↑ | 중 |
| 2 | 배지 획득 시 플로팅 애니메이션 | 감정 몰입↑ | 중 |
| 3 | 완료 시 마이크로 인터랙션 | 사용감 개선 | 하 |
| 4 | 그라데이션→단색 전환 | AI 느낌 감소 | 하 |
| 5 | 간격·정렬 체계 통일 | 일관성↑ | 중 |
